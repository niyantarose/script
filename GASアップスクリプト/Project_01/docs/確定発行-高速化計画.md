# 台湾 確定発行 高速化 計画（毎行フル読みの撲滅）

対象: `台湾書籍系_共通.js` の `台湾書籍系_ローカル確定発行_`（台湾まんが / 台湾書籍その他 共通）
前提: 2026-07-20 に safe win（skipFlush/列マップ/lastCol 共有）は反映済み（コミット f33af4f）。

## 根本原因（確定済み）
行エンジン `1行補完_共通_` が毎行 `同一作品情報を補完_` を呼び、その中で:
- `同一作品行から情報を取得_`（556行）: **商品シート全体を getValues**（同一作品の行を探すため）
- `Worksから同一作品情報を取得_`→`Worksから取得_`（450行）: **Worksシート全体を getValues**（タイトル候補ごと）

→ チェック N 行で「重い全読みが最大 2N 回」。これが支配的コスト。

## 方針（ロジック不変・最小リスク）
既存の `使用済みIDキャッシュ` と同じ**モジュールレベルのバッチ文脈**を導入し、確定発行の間だけ有効化する。
**onEdit や他の呼び出しは文脈を設定しない＝従来どおり毎回フル読み**なので挙動不変。

### バッチ文脈オブジェクト（module-level）
```js
let _TWBK_確定Ctx_ = null; // { sheetName, 列, sheetValues, worksSheetName, works }
function _TWBK_確定Ctx設定_(ctx){ _TWBK_確定Ctx_ = ctx; }
function _TWBK_確定Ctxクリア_(){ _TWBK_確定Ctx_ = null; }
```
- `sheetValues`: 商品シート 2..lastRow の 2D 配列。**書き込み追従**＝各行処理後にその1行だけ再読込して差し替え。
- `works`: Works の行配列（`Worksから取得_` が使う形）。**書き込み追従**＝新規Works作成時に push。

## 触る4点だけ
1. **`同一作品行から情報を取得_`（556）**: 冒頭で `_TWBK_確定Ctx_` が同じシートなら `getValues` の代わりに `_TWBK_確定Ctx_.sheetValues` を使う。無ければ従来通り。
2. **`Worksから取得_`（450）**: `_TWBK_確定Ctx_` があれば Works シート読取の代わりに `_TWBK_確定Ctx_.works` を走査。無ければ従来通り。
3. **`Worksを取得または作成_`（3261）**: 新規Works行を作成したら、同じ行データを `_TWBK_確定Ctx_.works` にも push（文脈がある時だけ）。※作成ロジック自体は不変、追記だけ。
4. **`台湾書籍系_ローカル確定発行_`（3032）**: 
   - `try` の直前で `_TWBK_確定Ctx設定_({...})`（商品シート全読み1回＋Works全読み1回）。
   - `対象行.forEach` の各 `1行補完_共通_` の**後**に、`sh.getRange(row,1,1,lastCol).getValues()[0]` でその1行だけ再読込し `sheetValues[row-2]` を差し替え（＝同一作品の後続行が最新IDを見られる）。
   - `finally` で `_TWBK_確定Ctxクリア_()`。

読み取り回数: **N（1行再読込）＋2（初回フル）** に激減（現状 2N フル読み）。

## 正しさ（最重要）
「同じ新規作品の複数対象行が同じ作品IDになる」不変条件を守る。
- 行N が作品IDを採番／Works新規作成 → sheetValues[N-2] 差し替え＆works push。
- 行N+1（同一作品）は文脈の最新 sheetValues/works を見て**同じID**を得る。

## テスト（tests/ の node ハーネスに追加：kakutei-batch-cache.test.js）
純粋ロジックを抽出してテスト（GASに依存しない形）:
- (a) 同一新規作品の2行 → 同一作品ID・Works追記1回。
- (b) 既存作品の行 → 文脈のworksから既存IDを拾う。
- (c) 文脈OFF（onEdit相当）→ 従来経路（フル読み）で同結果。
- (d) 別媒体（CM/NV）同名 → 別作品ID（媒体分離が壊れない）。

## ロールバック
文脈導入は加算的（無ければ従来経路）。問題時は `_TWBK_確定Ctx設定_` の呼び出し1行をコメントアウトすれば即・従来動作へ戻る。
