# 取り置き登録「出てくる注文は全行表示」設計 (2026-07-20)

## 背景

2026-07-20 に取り置き登録シートを「全行表示化」した（即納行を水色・表示専用で追加、
判断済み〈出荷済み/未着/予約〉も隠さない、記憶は棚確認セルへ一本化）。本番反映済み
（v3ブランチ 1bf0162）。

その運用中に**残る穴**が判明した：

- 実例: 受注 **10116625**（八谷 結）に、棚には来ている（部分在庫）のに
  **取り置き登録シートに行が出てこない商品**があり、数量を入れられなかった。
- 原因: その行は **受注ステータス「予約中」で入荷スタンプ（旧入荷日）が無い**ため、
  証拠ベースの絞り込み（`取り置き_登録絞り込み_`）で「まだ届いていない予約」と判定されて
  非表示になっていた。実際には早着して棚にあるのにスタンプが無いだけ。
- さらに `区分_` が `即納 / 取り寄せ / 指定なし` を返すうち、**指定なし**行は
  `取り置き初期登録を作成本体_` の収集ループで捨てられており、これも「入れる行がない」。

ユーザーの要望（原文趣旨）:
> このお取り置き登録シートに出てくる注文は、注文内容を即納でもお取り寄せでも全部出したほうがいい。
> 部分在庫になっているものは表示したい。ただし出荷GOはもう確定で荷物が届いているから表示不要。

## 目的

**このシートに1行でも出てくる注文（＝棚に何か来ている「アクティブ注文」）は、その注文の
受注明細を全行表示し、棚にある物はどの行でも数量入力できるようにする。** 箱（注文）と
シートを 1:1 で突き合わせられる状態にして、今回のような「入れる行がない」取りこぼしを無くす。

## 非目的（変更しないこと）

- **純粋な予約だけの注文（棚に何も来ていない）は従来どおり非表示。** 7/18 で確立した
  「証拠ベース表示（証拠が無い予約中は隠す）」の巻き戻しはしない。証拠ゲートは**注文単位**で
  維持する（注文が出るか出ないかの判定は今までどおり）。本設計が変えるのは「出てきた注文の
  “中”で、これまで隠していた行も出す」という**注文内の網羅性**だけ。
- 全件再計算・台帳の構造・反映ロジックは変えない。

## 仕様

### アクティブ注文の定義（変更なし）

受注番号（ban）が次のいずれかのソースに載る注文＝アクティブ:
入荷スタンプあり（着済スタンプ）／部分包装／④の各一覧（出荷可能・出荷GO未入金・部分在庫・
希望日待ち）。`取り置き_初期候補_` はこの ban 集合に載る注文行だけを候補化する（現状のまま）。

### 行の種類ごとの表示ルール

アクティブ注文の受注明細を全行走査し、種類ごとに次の扱いにする。

| 行の種類（判定元） | 扱い | 実装状況 |
|---|---|---|
| **即納**（`区分_`＝即納） | 水色 `#cfe2f3`・**表示専用**（現物取り置き数量は入力不可＝反映時エラー） | 実装済み(7/20) |
| **取り寄せ**（`区分_`＝取り寄せ） | **チェック可能**（現物取り置き数量を入力可）。**予約中で証拠なしも隠さない** | 本設計で変更 |
| **指定なし**（`区分_`＝指定なし） | **チェック可能**（実在する注文行なので拾えるように収集する） | 本設計で追加 |
| **出荷GO**（受注ステータスに「出荷GO」を含む） | **非表示**（従来どおり。確定・到着済み・発送作業中で棚確認不要） | 変更なし |
| **台湾/中国（別ルート）**（`引当_別ルート判定_`） | **対象外**（別在庫管理。混在は稀。必要になれば別途） | 変更なし |

### 変更点の要約

1. `取り置き_登録絞り込み_`: 「予約中で入荷証拠なし → 非表示」の除外を**やめる**。
   出荷GO の除外は**残す**。
   - 純粋予約注文はそもそも候補に入らない（アクティブ注文だけが候補）ので、この緩和で
     ノイズ（未着予約の大量表示）は増えない。影響は「アクティブ注文の中の予約中行」だけ。
2. `取り置き初期登録を作成本体_` の収集ループ: **指定なし行も `orders` に含める**
   （現状 `区分!=='取り寄せ'` で捨てているのをやめ、即納・別ルート・出荷GO 以外は拾う）。
   指定なし行の入荷スタンプも着済スタンプ集合に加える（その注文をアクティブ化する材料になる）。
3. 即納・出荷GO・別ルートの扱いは現状維持。

### 表示・入力の細部

- 予約中で証拠なしの取り寄せ行が出ても、`取り置き_棚確認判定_` は証拠が無いので
  **要棚確認（黄色）にはしない**＝普通の白い行として出る。受注ステータス列に「予約中」が
  見えるので状態は分かる。棚にあれば数量を入れる、無ければ空のまま（＝登録されない・安全）。
- 反映（`取り置き_統合反映計画_`）のガードは現状のまま:
  即納行に数量 → エラー、棚確認が出荷済み/未着/予約なのに数量 → エラー。予約中は受注ステータス
  であって棚確認ではないため、予約中行に数量を入れて登録するのは正常。

## テスト（TDD）

`tests/project24_reservation_workflow.test.js` に追加:

1. アクティブ注文の「予約中・入荷証拠なし」取り寄せ行が、`取り置き_登録絞り込み_` で
   落ちずに残る（従来は落ちていた）。
2. 「出荷GO」行は引き続き `取り置き_登録絞り込み_` で非表示。
3. 指定なし行が候補収集に含まれる（`取り置き初期登録を作成本体_` 相当の純粋ロジック、
   または収集ヘルパを切り出してテスト）。
4. 純粋な予約だけの注文（どのソースにも載らない）は候補に入らない＝非表示のまま
   （証拠ゲートが注文単位で効いていることの回帰テスト）。
5. 既存テスト（即納表示専用・判断済み全行表示・棚戻し・台帳確保重ね）は全てGREENのまま。
