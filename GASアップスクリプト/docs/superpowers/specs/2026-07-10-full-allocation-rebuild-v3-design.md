# 引当全件再構築 v3 設計補足

日付: 2026-07-10  
対象: Project_24  
基礎仕様: `2026-07-10-full-allocation-rebuild-design.md`

## 採用方針

ユーザーの再承認により、v2 の読み取り専用方式を廃止し、基礎仕様の「P列・引当履歴・韓国EMS商品の入荷日を事実データから全件再構築する」方式を採用する。

次の補足を基礎仕様へ追加する。

1. EMS番号が `棚卸` で始まる自動生成行は、実EMS供給として数えない。反映時はバックアップ後にステータスを `再計算除外` とし、数量・到着日・EMS番号は変更しない。
2. 発送実績は `発送済み` シートを直接読む。一意キーは `受注番号 + 商品ID`、同一キーは取込日時が最新の行を採用する。商品IDが違う分割発送は別実績として残す。
3. Yahoo末尾 `a` は未引当自由在庫の下限として確保する。EMS残数が「Yahoo a + 現在取寄せ需要」を超えるSKU、発送消費をEMSで説明できないSKU、照合が一意でないSKUは重要警告とし、自動反映対象から除外する。
4. 重要警告SKUはP列・履歴・受注明細の入荷日を作らず、通常の②引当でも在庫0として扱う。元データ修正後に再プレビュー・再反映するまで出荷可能にしない。
5. `棚卸箱をEMSリストへ追記` は誤在庫を作る旧方式として無効化する。
6. 反映ではプレビュー時のYahoo・EMS・発送済み・受注明細の署名一致を必須とし、引当ファイルと発注共有ファイルの両方をバックアップする。

## 完了条件

- 合成棚卸行を供給に含めない。
- GoQ発送済みの重複スナップショットを1件として扱う。
- 発送日より後のEMSを発送済みへ割り当てない。
- 現在取寄せは注文日時の古い順に割り当てる。
- Yahoo自由在庫を侵食しない。
- 重要警告SKUは出荷可能にならない。
- 台湾・中国ルートの手入力入荷日は保持する。
- 実データ反映前にプレビューと2ファイルのバックアップがある。
