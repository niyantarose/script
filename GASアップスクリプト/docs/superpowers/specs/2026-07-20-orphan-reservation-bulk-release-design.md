# 孤児取り置き一括解除ツール 設計 (2026-07-20)

## 背景

品切れ→代替品で発送された注文は、受注明細から消える一方、取り置き台帳には元商品の「取り置き中」が
残る（例: 10117492 MOFUN-MNN-02）。②引き当て実行のたびに「取り置き台帳に現注文の無い行(手動解除して
ください)」の**タグ注意**が出る（[取り置き計算.js:411](../../../Project_24/取り置き計算.js)）。
現状の解消は「取り置き台帳タブで行を選択→🔧選択した取り置きを手動解除」の1行ずつ手作業のみ。

代替発送は**たまに**起きる。孤児の判断（代替かどうか）は人がやる前提で、**解除の手間だけ減らす**補助ツールが欲しい。

## 目的

孤児（取り置き中だが受注明細に注文が無い取り置き）を一覧化し、チェックで選んでまとめて手動解除できる
メニューを追加する。既存の孤児検知ロジックと同じ基準で拾い、消込台帳の出荷形跡を添えて人の判断を助ける。

## 仕様

**メニュー**: 📥受注・共通 →「🔧 取り置き内部管理」に「🧹 孤児取り置きをまとめて解除」を追加。

**流れ**:
1. 台帳をスキャンして孤児を抽出（下記ロジック）。0件なら「孤児はありません」で終了。
2. チェックボックス付きモーダルダイアログ（ダニエル引当と同じHTML方式）を表示。
   各行: 受注番号 / 商品コード / 数量 / 出荷形跡（あり・なし）。**出荷形跡あり=既定チェック済み**、なし=オフ。
   解除理由の入力欄（既定「品切れ・代替品で発送」）。
3. 「解除」で、選ばれた取置IDの「取り置き中」行だけを状態=手動解除にして一括保存。
4. 完了トースト（解除件数）。②を回すとタグ注意が消える。

**判定ロジック**:
- **孤児** = 台帳の `状態===取り置き中` かつ 行キー `取り置き_行キー_(受注番号|商品コード|SKU)` が
  今の受注明細の全行キー集合に**無い**もの。
- **出荷形跡** = その受注番号が消込台帳の出荷済み行（`消込台帳_出荷済み行_()` の ban）に載っている。
  代替でも受注番号一致で「形跡あり」＝既定チェックにする。

**安全策**:
- 対象は「取り置き中」だけ（発送済み・戻し・既に手動解除済みは触らない＝冪等）。
- 解除理由は必須。
- 選択0件・理由空は実行しない。
- 書き込みは `直列_`（DocumentLock）でラップ。

## 作り（テスト可能に分離）

- 純粋関数 `取り置き_孤児取り置き一覧_(ledger, 注文キー集合, 出荷済み受注番号集合)`
  → `[{取置ID, 受注番号, 商品コード, SKU, 取り置き数量, 出荷形跡}]`
- 純粋関数 `取り置き_一括解除計画_(ledger, 解除ID集合, 理由, now)` → 解除後のledger行
  （指定IDの取り置き中だけ状態=手動解除・終了理由メモ=理由・更新日時=now、他は不変）
- GASアダプタ `孤児取り置きをまとめて解除()`（読み+ダイアログ表示）と
  `孤児取り置き一括解除実行_(ids, reason)`（`直列_`で保存）。ダニエル引当のダイアログ方式を踏襲。

## テスト（`tests/project24_reservation_workflow.test.js`）

1. `取り置き_孤児取り置き一覧_`: 注文に有る取り置き中は出さない／注文に無い取り置き中だけ出す／
   発送済み・手動解除は無視／出荷形跡フラグが受注番号一致で立つ。
2. `取り置き_一括解除計画_`: 指定IDの取り置き中→手動解除(理由・更新日時)、非指定は不変、
   取り置き中以外の指定IDは触らない（冪等）。
