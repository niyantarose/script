# 受注明細から直接 個別引当/キャンセル（個別対応シート運用の廃止）

日付: 2026-07-08 / 対象: Project_24（引当ファイル）

## 背景・課題

個別対応シート運用は「照合キーの転記」「操作列＋ボタン2つの二度手間」「14列のシート」「実行結果を消さないと再実行できない作法」が重い（ユーザー確認済み・全項目が不満）。

## 決定

受注明細で行を選択し、シート上部の**ボタン画像**（個別引当＝緑／引当キャンセル＝赤）を押すだけの運用に変える。
メニューではなくボタン画像（ユーザー指定）。個別対応シートはメニューから撤去し、シート自体も削除してよい（コードは残置、`個別対応シートを作成`でいつでも復元可能。実行ログは引当履歴シートが正）。

## 動作仕様

### 個別引当（選択行ごと）
1. 受注明細のアクティブ選択から対象データ行を取得（複数行可・ヘッダー以下のみ）
2. 取り寄せ行のみ対象。受注番号＋商品コード/SKUから、発注共有EMSリストの**到着済**行を照合（`受注候補コード_`→`codeKeys_`、②と同じルール。照合キー入力は不要）
3. 箱の残数＝行数量−P列既存割当。残ありの箱だけ候補
4. 候補1つ→確認ダイアログ／複数→番号選択（到着日・EMS番号・Box・残数を表示）→確認
5. 実行: P列へ追記（`個別対応_P注文整形_`、分割は薄黄）＋`引当履歴_個別記録_`＋入荷日が空欄ならEMS到着日を記入（既に入っていれば変更せず通知）
6. 二重割当ガード: 書き込み直前にP列セルを読み直す（スナップショットの古さで上書きしない）。同じ箱へ既に注文個数分の割当があればスキップ

### 引当キャンセル（選択行ごと）
1. EMSリスト全行から「この受注番号×この商品」のP列割当を検索（状態問わず）
2. 合計個数と内訳を確認ダイアログ→OKで全削除（P列書き換え＋`引当履歴_キャンセル_`）
3. `個別対応_入荷日クリア_`で、他に有効割当が無ければ受注明細の入荷日をクリア

### ボタン
- 受注明細の1行目（凡例より上・フリーズ域内）にSVG画像ボタン2つ（`assignScript`）。タイトル接頭辞`受注個別ボタン_`で識別し、onOpenで毎回貼り直し
- 画像クリックは選択範囲を動かさないので「行を選択→ボタン」が成立する
- 再設置用メニュー「🎯 個別ボタンを設置(受注明細)」を1項目だけ残す

### メニュー変更（引当.js onOpen）
- 撤去: 個別対応シートを作成／照合キーで補完／個別受注引当／引当キャンセル（4項目）
- onOpenの`個別対応ボタンを設置_(true)`→`受注明細個別ボタンを設置_(true)`に差し替え
- `onSelectionChange`の個別対応セルボタン処理は残置（シートが無ければ即false・無害）

## 実装方針

- 新規ファイル `Project_24/受注明細個別ボタン.js` のみ追加。P列展開/整形・履歴記録/キャンセル・入荷日クリアは個別対応.jsの既存関数を再利用（実績ロジックの使い回し）
- 候補抽出＋残数計算は純関数`受注個別_候補_`に切り出し、Nodeテストで検証してからデプロイ

## 復元手段（ロールバック）

- シート: `個別対応シートを作成`を実行すれば再作成（メニュー撤去後もApps Scriptエディタ/関数から実行可）
- メニュー・ボタン: git revertでonOpenを戻すだけ
