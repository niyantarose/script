# 仕入れ計算機 v2（台湾・韓国・中国）設計書

作成日: 2026-07-16

## 目的

仕入れ計算機を、手数料・送料・消費税をいつでも変えられ、
うわのせを微調整しながら「いくら残るか」と円単位の売値（例: 8480）まで見える形にする。

対象は台湾・韓国・中国のすべて。シートとサイドバーを同仕様にする。

## 背景 / 現状の問題

現在の `台湾計算機` / `韓国計算機` / `中国計算機` およびサイドバーは次が不足している。

1. 送料率・モール手数料・消費税が数式に直書き（`0.35` / `0.09` / `0.1` 等）で変更しづらい
2. うわのせを手で微調整して、残り粗利額・実質粗利率を見る欄がない
3. おすすめ売値が100円丸めのみで、理論上の円単位売値（8480 等）が見えない

古い `台湾価格計算表` / `中国価格計算` 等は残し、触らない。

## スコープ

### 含む

- `GASアップスクリプト/Project_01/計算機.js`（きれいな計算機シート再生成ロジック）

## 共通レイアウト（3国）

### セクション構成

1. **入力（黄色）** — いつでも変更可
2. **自動計算（グレー）** — 原価など
3. **結果（緑）** — 理論売値・丸め売値・推奨うわのせ・微調整結果・粗利・判定

### 入力項目

| 項目 | 台湾 | 韓国 | 中国 | 初期値（目安） |
|---|---|---|---|---|
| 商品価格 | 元 | ウォン | 元 | 現状どおり |
| 重さ(g) | — | ○ | — | 300 |
| 目標粗利率 | ○ | ○ | ○ | 0.25 / 0.25 / 0.30 |
| 使用レート | ○ | ○ | ○ | 市場レート参照（上書き可） |
| 市場レート(自動) | ○ | ○ | ○ | GOOGLEFINANCE / API |
| 送料率 | 0.35 | — | 0.30 | 入力可 |
| 送料計算 | 価格×送料率 | 重さ×0.6（円） | 価格×送料率 | — |
| モール手数料 | 0.09 | 0.09 | 0.09 | 入力可 |
| カード手数料 | — | — | 0.03 | 入力可 |
| 消費税 | 0.10 | 0.10 | 0.10 | 入力可 |
| **実際のうわのせ（微調整）** | ○ | ○ | ○ | 推奨うわのせを初期値に |

台湾・中国の「送料」は、送料率から自動計算した額を表示しつつ、
送料率セルを変えれば即反映する。送料額そのものを手入力したい場合は、
送料率の代わりに「送料(元)」を黄色入力にし、率は参考表示でもよい。
**採用: 送料率を黄色入力、送料額は自動計算表示**（現状運用に近い）。

韓国は従来どおり `重さ(g)×0.6` を送料相当とし、係数 `0.6` も黄色入力にする。

### 手数料の扱い

実質コスト率:

```
feeTotal = モール手数料 + カード手数料(中国のみ) + 消費税
```

台湾・韓国はカード手数料なし（0）。中国はモール 0.09 + カード 0.03 = 0.12 が従来の合算に相当。
内訳を分けて入力できるようにし、合算は数式で行う。

### 計算式

記号:

- `C` = 原価(円)
- `g` = 目標粗利率
- `f` = feeTotal
- `U` = 実際のうわのせ（微調整入力）

```
理論売値 S_theory = C / (1 - g - f)
おすすめ売値 S_round = ROUND(S_theory, -2)   // 100円単位
推奨うわのせ U_rec = S_round - C

実際売値 S_actual = C + U
実質粗利額 P = S_actual - S_actual * f - C
実質粗利率 r = P / S_actual
```

判定: `r < 0.2` なら警告、それ以外 OK。

### 結果に必ず出す項目

1. 原価(円)
2. **理論売値**（円単位・丸めなし、例: 8480 / 8509）
3. **おすすめ売値（100円丸め）**
4. 推奨うわのせ
5. 実際のうわのせ（入力）
6. 実際売値
7. 実質粗利額（いくら残るか）
8. 実質粗利率
9. 判定

サイドバーも同じ項目を表示する（入力は黄色相当、結果は緑相当の見た目）。

## シート再生成の振る舞い

- メニュー「✨ きれいな計算機シートを作成」実行時:
  - 同名シートがあれば削除して作り直す（現行どおり）
  - 古い `台湾価格計算表` 等は削除しない
- 再作成時の「実際のうわのせ」初期値は、推奨うわのせと同じ値を数値で入れる
  （数式参照だと微調整しづらいため、初期は値コピー。運用上は推奨値を見て手で合わせる）

補足: シート上で推奨うわのせが変わったとき、実際のうわのせを自動追従させると
微調整が上書きされる。**自動追従はしない**（明示的にコピーする運用）。

## サイドバー

- 3国タブ共通 UI
- 送料率（または韓国の重量係数）・モール手数料・カード手数料（中国）・消費税を入力可能
- 実際のうわのせ入力 → 実際売値・粗利額・粗利率を即再計算
- 理論売値と100円丸め売値の両方を表示
- レート自動取得は現行 `getKeisanRates` を継続

## 成功条件

- 台湾・韓国・中国で、手数料・送料（率/係数）・消費税を黄色欄から変更できる
- うわのせ微調整で、実際売値・残り粗利額・実質粗利率が更新される
- 理論売値（円単位）とおすすめ売値（100円丸め）の両方が見える
- 既存メニューから再作成・サイドバー起動できる
- 古い価格計算表シートは破壊しない
