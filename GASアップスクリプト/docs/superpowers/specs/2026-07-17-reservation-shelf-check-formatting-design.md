# 取り置き登録・棚確認表示改善 設計書

**作成日:** 2026-07-17  
**対象:** Project_24「取り置き登録」シート  
**状態:** ユーザー承認済み

## 目的

取り置き登録の棚確認作業で、同じ注文に属する商品を見分けやすくし、各商品行の棚確認結果を色で即座に判別できるようにする。また、未来発売の予約商品は候補から除外しつつ、商品名に残っている古い「予約」表記によって、すでに入荷している商品が候補から消える問題をなくす。

## 現状と問題

- 商品名・選択肢の「予約」表記だけを用いた自動判定が、発売・入荷済みの商品まで予約扱いにすることがある。
- 棚確認の候補行が注文単位で視覚的に区切られていない。
- 棚確認プルダウンを変更しても、行の見た目が変わらず判断結果を確認しにくい。
- Claude Codeの途中差分では予約表記の自動判定撤去まで実装されているが、既存テストが旧仕様のまま2件失敗している。

## 採用する方式

### 1. 候補からの自動除外

自動除外は受注ステータスと、商品名・選択肢から読み取れる未来発売日に限定する。

- `予約中`を含む受注ステータス: 旧入荷日・部分包装・数量入力の有無にかかわらず除外
- `出荷GO`を含む受注ステータス: 旧入荷日・部分包装・数量入力の有無にかかわらず除外
- 商品名または選択肢に「予約」があり、発売予定日が現在より未来と判定できる: 除外
- 予約表記の日付が過去: 表示
- 「予約早期完売」など日付を読み取れない予約表記: 安全のため表示して人が棚確認
- 上記以外: 表示

商品名または選択肢に「予約」「発売予定」などが含まれるだけでは除外しない。未来日と判断できた場合だけ内部的に予約扱いとし、旧入荷日スタンプがある行または受注ステータスが `部分包装` の行は現物の形跡を優先して表示する。この現物形跡の例外は「商品名・選択肢から判定した未来予約」にだけ適用し、受注ステータスが `予約中` / `出荷GO` の行には適用しない。

### 2. 手動の「予約」判定

棚確認プルダウンには次の選択肢を残す。

- 発送待ち
- 部分在庫
- 出荷済み
- 未着
- 予約

`出荷済み`、`未着`、`予約`を選び、現物取り置き数量が空欄の行は「判断済み」とする。

- 選択直後は行を残し、条件付き書式の色で選択結果を確認できる。
- 次回「取り置き登録を作成」を実行したとき、判断をDocumentPropertiesへ保存して一覧から非表示にする。
- 注文自体が候補から消えた場合は、その取置IDの保存済み判断も自動削除する。
- 現物取り置き数量が入っているのに `出荷済み`、`未着`、`予約`が選択されている場合は矛盾として確定を止める。

### 3. 注文ごとの太い罫線

取り置き登録を作成した後、データ範囲には薄い灰色の格子を設定する。そのうえで、受注番号が次の行で変わる位置と最終行に黒い太い下罫線を設定する。

- 罫線判定キー: `受注番号`
- 色: 黒
- スタイル: `SOLID_THICK`
- 対象: 取り置き登録の全列

既存の受注明細用 `注文罫線_` / `注文境界A1_` と同じ境界計算を再利用し、行ごとのAPI呼び出しは行わない。

### 4. 棚確認ステータスによる行色

条件付き書式を使い、棚確認セルを変更した直後に、選択した商品行だけを色付けする。同じ受注番号の他商品行には色を広げない。

| 棚確認 | 行の背景色 | 補足 |
|---|---|---|
| 発送待ち | 薄い青 `#cfe2f3` | 発送作業待ち |
| 部分在庫 | 薄い緑 `#d9ead3` | 現物の一部あり |
| 出荷済み | 赤 `#f4cccc` | 濃い赤文字・太字も設定し最優先で目立たせる |
| 未着 | 灰色 `#d9d9d9` | 現物なし |
| 予約 | 薄い紫 `#d9d2e9` | 手動で予約と判断 |
| 空欄かつ要棚確認 | 黄色 `#fff2cc` | 従来の注意表示を維持 |

条件付き書式の適用範囲はデータ行の全列とし、数式は棚確認列の見出し位置から動的に作る。取り置き登録はスクリプト管理の生成シートであるため、作成時にこのシートの条件付き書式を上記ルールへ更新する。

## 処理の流れ

1. 受注明細と各派生一覧から候補を集める。
2. 同じ受注番号・商品を既存ルールでまとめる。
3. 既存の手入力数量、棚確認、メモを引き継ぐ。
4. 最初に、受注ステータスが `予約中` / `出荷GO` の候補を無条件で除外する。次に、商品名・選択肢から未来発売と確認できる予約候補を除外する。ただし後者だけは、旧入荷日または `部分包装` の現物形跡がある場合に表示する。
5. 保存済みの `出荷済み`、`未着`、`予約`判断を適用し、判断済み行を非表示にする。
6. 要棚確認を上へ並べ替えてシートを一括保存する。
7. 棚確認プルダウン、黄色の基本背景、注文境界の太線、ステータス条件付き書式をまとめて設定する。

## 安全性と性能

- 引当数量、EMS供給、取り置き台帳の計算式は変更しない。
- 表示候補・書式・手動判断の保存だけを変更対象にする。
- `onEdit`へ新しい処理は追加せず、色変更はGoogle Sheetsの条件付き書式に任せる。
- 罫線はRangeList、背景は一括設定、条件付き書式は固定本数で設定し、1行ずつ書き込まない。
- Claude Codeが残した未コミット差分を基点とし、他タスクの差分を上書きしない。

## テスト方針

先に旧仕様テストを新仕様へ直し、必要な失敗テストを追加してから本体を変更する。

1. 商品名・選択肢の未来発売予約だけを自動判定し、過去日・日付不明は自動除外しない。
2. 受注ステータス `予約中` と `出荷GO` は商品名に関係なく除外される。
3. `予約受付終了`など、未来日を含まず受注ステータスも `予約中` ではない文字列は除外しない。
4. 旧入荷日または `部分包装` がある行は、商品名・選択肢由来の未来予約表記より現物形跡を優先する。
5. `旧入荷日 + 予約中`、`部分包装 + 予約中`、`部分包装 + 出荷GO` は受注ステータスを優先して除外する。
6. 手動 `予約` は `出荷済み`・`未着`と同様に記憶され、次回更新で非表示になる。
7. 同じ受注番号の連続行の末尾にだけ太い境界が設定される。
8. 棚確認5種類に対応する条件付き書式と色が設定される。
9. `出荷済み`は赤背景・濃い赤文字・太字になる。
10. 候補数が減少または0件になったとき、前回データ行の背景色・格子・太線が空行へ残らない。
11. 既存の取り置きワークフロー全テストとProject_24全関連テストが通る。

## 対象外

- 引当計算ロジックの変更
- Yahoo在庫数・EMS在庫数の再計算
- 受注明細の商品名や選択肢の書き換え
- 棚確認を選択した瞬間の行削除
- 本変更と無関係なサーバー在庫ツールの未コミット差分
