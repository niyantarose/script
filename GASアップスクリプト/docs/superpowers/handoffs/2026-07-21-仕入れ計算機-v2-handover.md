## 1. これは何か

仕入れの売値・粗利を、台湾／韓国／中国ごとに計算するツール。

| 入口 | 内容 |
|---|---|
| メニュー「💴 計算機」→「🧮 電卓を開く」 | サイドバー電卓 |
| 同 →「✨ きれいな計算機シートを作成」 | `台湾計算機` / `韓国計算機` / `中国計算機` を再生成 |
| 同 →「🩺 動作確認（軽い）」 | toast のみ。GASが生きてるか確認用 |

**触ってよいシート名（新）:** `台湾計算機` / `韓国計算機` / `中国計算機`  
**触らない（旧）:** `台湾価格計算表` / `中国価格計算表` / `韓国価格計算表` など  
スクショで「台湾価格計算表」を開いている場合、計算機メニューの成果物ではない。タブに「台湾計算機」が出るかで成否を見る。

## 3. 仕様の正（実装が設計書より進んでいる）

元設計: `docs/superpowers/specs/2026-07-16-仕入れ計算機-v2-design.md`  
※設計書は古い（理論売値・GOOGLEFINANCE・delete作り直し・原価帯はメモ程度、など）。**実装の方が正**。

### レイアウト順（シート・サイドバー共通）

1. **入力（黄）** — 毎回触る欄  
2. **微調整結果（青）** — 実際売値・実質粗利額（大きく）・実質粗利率・判定  
3. **おすすめ（緑）** — **10円丸め** と **100円丸め**（理論売値は廃止）  
4. **自動価格計算結果（ピンク）** — 原価帯売値 10円 / 100円のみ  
5. **自動計算（灰）** — 手数料合計・送料・原価・適用目標粗利率  
6. **【別設定】原価帯別目標粗利率（最下・黄帯）** — ユーザーが帯ごとの率を設定

### 原価帯と目標粗利率

- 帯ごとの率はユーザーが設定する。
- 原価に応じて「適用目標粗利率」が自動。
- 黄色の「目標粗利率」は通常 `=IFERROR(適用セル, …)` で連動。手で数値上書き可。
- 帯の初期値: 〜2千35% / 〜6千25% / 〜1.2万20% / 〜2万10% / 超8%。

### 計算の要点

```
feeTotal = モール + カード(中国のみ) + 消費税
おすすめ売値 = ROUND(原価 / (1 - 目標粗利率 - feeTotal), -1 or -2)
実際売値 = 原価 + 実際のうわのせ
実質粗利額 = 実際売値 - 実際売値×feeTotal - 原価
判定: 実質粗利率 < 20% なら警告
```

- うわのせ初期値はおすすめ(10円丸め)基準の数値コピー。自動追従しない。
- レート: シートは数値の目安（GOOGLEFINANCE依存を外した）。サイドバーは `getKeisanRates`（open.er-api.com）。

### 国ごとの原価

| 国 | 原価 |
|---|---|
| 台湾 | `(価格元 + 送料元) × レート`、送料元=`価格×送料率` |
| 中国 | 同上＋カード手数料あり |
| 韓国 | `価格ウォン×レート + 重さ(g)×重量係数` |

1. **古いタブを見ている** → 新シート名は `〇〇計算機`。

## 8. 会話で確定した要望メモ（設計書に無い／上書き）

- 黄色入力は毎回使う欄（価格・レート・手数料・目標粗利率・うわのせ）。
- 原価帯の利益率は【別設定】でユーザーが決め、原価に応じて自動適用。

- ピンク枠は原価帯売値だけ。手数料・原価は下の自動計算へ。
- 実質粗利額も実際売値と同様に大きく。
- サイドバーのおすすめ(10円)も100円と同じ大きな表示（反映済み想定）。
