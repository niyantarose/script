## 0) 現状確認


## 1) DBバックアップ（必須）

```bash
# SQLite の場合
cp instance/inventory.db "instance/inventory.backup-$(date +%Y%m%d)-pre-ledger.db"

# PostgreSQL の場合（.env の DATABASE_URL を使用）
pg_dump "$DATABASE_URL" > ~/zaiko-backup-$(date +%Y%m%d)-pre-ledger.sql
```

## 2) コード更新 + 再起動

```bash
git pull
sudo systemctl restart zaiko-tool
systemctl status zaiko-tool --no-pager | head -5
```

新テーブル（stock_transactions / mall_skus）は起動時の create_all で自動作成される。マイグレーション作業は不要。

## 3) 即納の重複行チェック（Step 0）

```bash
cd /home/ubuntu/zaiko-tool/app
python3 - <<'PY'
import sys; sys.path.insert(0, '.')
from app import app
from models import db
from sqlalchemy import text
with app.app_context():
    rows = db.session.execute(text(
        "SELECT product_code, COUNT(*) FROM inventory "
        "WHERE inventory_type='即納' GROUP BY product_code HAVING COUNT(*)>1"
    )).fetchall()
    print('即納の重複商品コード:', rows if rows else 'なし')
PY
```

重複が出た場合は、どの行が正か決めて1行に統合（quantityを合算し他行を削除）してから次へ。

## 4) シード実行（期首残高 + 未出荷注文バックフィル + YahooモールSKU）

## 6) 実棚突合（重要・手作業）

即納の quantity は旧システムで入荷加算のみ（出荷減算なし）だったため、**累積値の疑いが強い**。
主要商品について実際の手元在庫と台帳画面（/ledger/）の数を突き合わせ、差異は「±」→「調整」で訂正してから本運用とする。
