# 台湾まんが 日本語タイトル候補選定の改善 設計書

作成日: 2026-07-16

## 目的

博客來（books.com.tw）の繁体字商品名から日本語まんが原題を照会するとき、
**中国語タイトルを「日本語タイトル」として誤採用しない**ようにする。

例:

| 入力（繁体字） | 誤結果（現状） | 期待結果 |
|---|---|---|
| 披著狼皮的羊公主 | 披著狼皮的羊 | 狼の皮をかぶった羊姫 |

## 背景 / 問題

MangaUpdates でシリーズ一致自体は取れているが、日本語候補の選定が次の理由で失敗する。

1. `hasJapaneseTitleSignal` が「漢字があれば日本語」とみなすため、繁体字タイトルも候補に入る
2. シリーズ一致後（`seriesVerified`）は Associated Names の先頭を優先する
3. 原題エコー除外が **クエリとの完全一致（＋一部字体差）のみ** のため、
   `披著狼皮的羊公主` に対する `披著狼皮的羊` のような部分一致中国語が残る
4. かな入りの正しい日本語題（`狼の皮をかぶった羊姫`）より先に中国語が採用される

## スコープ

### 含む

- `core/mangaUpdatesClient.js` の日本語候補選定ロジック
  - 主に `pickPreferredJapaneseCandidate` / 必要ならその呼び出し側
- 同ロジックのユニットテスト追加（`tests/manga-updates-client.test.js`）

### 含まない

- UI / popup / GAS 連携の変更
- MangaUpdates / AniList の検索クエリ生成の変更
- 台湾タイトル別名辞書の拡充
- 案2（採用後バリデーション強化・フォールバック強化）

## 設計

### 変更の中心

候補選定の優先順を次に固定する。

1. Associated Names 等から候補を収集（現状どおり）
2. **中国語エコーを除外**
   - クエリとの正規化キー完全一致
   - クエリとの部分一致（短い方が長い方に含まれる、長さ閾値あり）
   - 既存の簡繁字体差正規化（`來/来`・`當/当` 等）もエコー判定に適用
3. **かな入り候補があれば、その中から先頭を採用**
4. かなが無いときだけ、漢字のみ候補を採用（K-9 のようなカナ無し日本語題を維持）
5. 残候補が無ければ空文字（中国語を「解決」結果に入れない）

### 部分一致エコーの判定方針

- 正規化後キーで、候補がクエリの部分文字列、またはクエリが候補の部分文字列
- 誤除外を抑えるため、短い方の長さが一定以上（既存の部分一致しきい値に合わせて **4文字以上**）のときだけエコー扱い
- 日本語かな入りタイトルはエコー除外の対象外（部分一致判定の前、またはかなシグナルでスキップ）

### 互換性（壊してはいけない既存ケース）

| クエリ | 期待日本語題 |
|---|---|
| 全球高考 | パニックテスト～これでも冷静でいられますか？～ |
| 螢幕情緣 | モニターごしの恋 |
| 今生我来當家主 | 今世は当主になります |
| K-9 警視廳公安部公安第9課異能對策組 | K-9 警視庁公安部公安第9課異能対策係 |

### 新規回帰ケース

Associated Names に次があるとき、クエリ `披著狼皮的羊公主` は
`狼の皮をかぶった羊姫` を返す（`披著狼皮的羊` を返さない）。

```
associated:
  - 披著狼皮的羊
  - 披著狼皮的羊公主
  - Sheep Princess in Wolf's Clothing
  - 狼の皮をかぶった羊姫
```

## エラー / 空結果の扱い

- 日本語候補が残らない場合は `japaneseTitle: ''`
- シリーズ一致自体は維持してよい（`series_found_no_japanese` など既存ステータスに委ねる）
- 中国語を埋めて「解決」扱いにしない

## テスト

`tests/manga-updates-client.test.js` に上記新規ケースを追加し、既存ケースがすべて通ることを確認する。

## 成功条件

- `披著狼皮的羊公主` 系の誤採用が解消される
- 上記互換ケースがすべて回帰しない
- 変更は候補選定に限定され、検索・UI・GAS に手を入れない
