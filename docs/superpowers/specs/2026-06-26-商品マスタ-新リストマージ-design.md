# 商品マスタ：新リスト（Daegu-Yamaguchi-Order）マージ機能 設計

日付: 2026-06-26
対象: GASアップスクリプト/Project_19（発注共有ファイル）

## 目的

別ファイル `Daegu-Yamaguchi-Order`（`.xlsm`）の `list` シートを「新しいおおもとのマスタ」として、
発注共有ファイルの **商品マスタ** へマージする。品目は英語に統一し、EMS発送で使える状態にする。
既存の「発注リスト大邱データ → 商品マスタ自動追加／重量などの差分補完」運用は変更しない。

## 前提（既存構造）

商品マスタ（HEADER_ROW=6, データは7行目〜）:
- A: No.（連番）/ B: 商品コード / C: 業者 / D: 商品名 / E: 品目 / F: 価格 / G: 重さ

新リスト `list`（ヘッダー行にラベル、データはその下）:
- `NAME`=商品名 / `CODE`=商品コード / `ITEM TYPE`=品目（英語）/ `price`=価格
- 業者・重量の列は無い。同じコードで名前違いの行が実在する（例: `essay0`, `sphoto004` 各2行）。

## 決定事項

1. **マージ方式**: 既存マスタは全て残し、新リストを足す（完全置換しない）。
2. **被りコードの上書き範囲**: 品目・商品名・価格 = 新リストで上書き。業者・重量 = 既存キープ。
3. **データの渡し方**: 発注共有ファイルに「新マスタ取込」タブを作り、`list` の中身をコピペ。メニューから実行。
4. **品目の英語統一**:
   - 新リストにある商品: ITEM TYPE（英語）をそのまま採用。
   - 旧マスタだけに残る行: 「品目マッピング」タブで英語へ変換（後述）。

## コンポーネント

### 1. 取込タブ「新マスタ取込」
- ユーザーが `list` の中身を貼り付ける。
- スクリプトは**ヘッダー名で列を特定**（`NAME` / `CODE` / `ITEM TYPE` / `price`）。位置ズレ・余計な列（1/UNIT/列）は無視。

### 2. 品目マッピングタブ「品目マッピング」
- A列=元の品目、B列=英語品目。
- ドライラン時、変換できなかった品目を**自動でA列に追記**（B列は空）。ユーザーがB列を埋める。
- 英語の語彙は新リストの ITEM TYPE 集合（Book / Picture Book / Textbook / Comic Book / Photo Book / Coloring Book …）を基準。

### 3. マージ関数 `商品マスタ_新リストをマージ()`（メニュー: Excel同期）
処理順:
1. 取込タブを読み、`{code, name, item(英語), price}` を作る。`normCode_` で正規化し、**取込内の重複コードは先勝ち**（残りはログ報告）。
2. 商品マスタを `normCode_(商品コード)` でインデックス化。
3. 品目マッピングを辞書化 `{元品目: 英語}`。英語集合 = 取込の ITEM TYPE distinct。
4. 変更を算出:
   - 取込コードが**マスタに有る** → D商品名・E品目・F価格 を取込値で上書き。C業者・G重量は既存維持。
   - 取込コードが**マスタに無い** → 新規行追加（B/D/E/F を埋め、C業者・G重量は空、A No.は連番継続）。
   - **マスタだけに有る行** → E品目を英語化: 既に英語ならそのまま／マッピングに有れば変換／無ければ元のまま残し「未マッピング」として収集。
5. **ドライラン確認ダイアログ**: 追加◯ / 更新◯ / 変更なし◯ / 取込内重複◯ / 未マッピング品目◯件（一覧）。未マッピング品目は品目マッピングA列へ追記。
6. YES → **商品マスタを複製バックアップ**（`商品マスタ_backup_yyyyMMdd_HHmm`）→ 変更を一括適用 → 実行ログに詳細記録。NO → 中止（マッピングタブへの追記だけ残る）。

## 安全策
- 実行前ドライラン確認（件数表示）。
- 適用直前に商品マスタを複製バックアップ。
- `Logger.log` に追加/更新/重複/未マッピングを記録。
- `normCode_` による表記ゆれ吸収（既存の照合と統一）。

## 既存運用（変更なし）
- 発注リスト大邱データで商品コード入力 → 商品マスタから業者/商品名/品目/価格/重さを自動補完（`autofillHachuByCfg_`）。
- 新コードは発注から商品マスタへ自動追加（マスタ登録）。
- 重量などの差分補完（`MASTER_SYNC_FIELD_RULES`）はそのまま。新リストに重量が無い分は、引き続き発注側から入ったときに補完される。

## スコープ外（今回やらない）
- 今後の発注追加分の品目を自動英語化（変換マッピングを同期に組み込む）。必要になれば別途。
- 別ファイルの直読み（openById）/ CSV取込。取込タブ方式に統一。

## 未確定（実装時にユーザー入力が必要）
- 品目マッピングB列の英訳（ドライランが未マッピング品目を出した後に記入）。
