# 台湾CN → Yahoo商品登録テンプレ 送信ボタン 設計書

作成日: 2026-06-19

## 目的

`商品登録よろしく _台湾CNなど` の各タブで、A列チェックボックスをONにした行を、
ボタン1つで別ブック `Yahoo商品登録テンプレ` の「①商品入力シート」へ転送する。

## 対象ブック / シート

- **ソース**: `商品登録よろしく _台湾CNなど`
  - spreadsheetId: `1OSoZnNTMHrH5YgU-j7zwOsyQsjS6bcwjfZxc3kzeKWM`
  - バインドGASプロジェクト: Project_01（scriptId `15o-6H_Hi5Ttx_XlE_A9HfGsjLdH5sbE2hXH7zzxtqkvEkj1DKvqCz0m4`）
  - 横断する台湾系タブ: **台湾まんが / 台湾書籍その他 / 台湾グッズ / 台湾雑誌**（設定定数で持つ）
- **送信先**: `Yahoo商品登録テンプレ`
  - spreadsheetId: `1jN-3aLsLg_wRjLQU-mDqaImVVBgdpvwJWsHegxSJRF0`
  - タブ名: **①商品入力シート**
  - ヘッダー: **1〜2行目**（2行ヘッダー） / データ: **3行目から** / 追記位置: 既存データの**最終行の下**

## 起動方法

- 各台湾系タブに**図形ボタン**を配置し、関数 `台湾_Yahooテンプレへ送信()` を割り当てる
  （図形はシートごとに手動配置。関数自体は全タブを横断するのでどのタブのボタンからでも同じ動作）

## 行の選択

- 各タブの **A列チェックボックス = TRUE** の行のみ対象
- A列が発番発行（チェックボックス）列である前提。実装時に各タブのヘッダー行から
  チェックボックス列（A列固定 or ヘッダー名）を確認する

## 列マッピング（タブ差をヘッダー候補名で吸収）

各タブの**ヘッダー行を実行時に読み**、Yahoo各列に対応するソース列を候補名で特定する。
送信先も①商品入力シートの1〜2行目を読み、ヘッダー名で書込列を特定する。

| Yahoo列（①商品入力シート） | ソース候補ヘッダー名 |
|---|---|
| 商品名 | タイトル / 商品名（出品用） / 雑誌名 |
| 商品コード | 親コード / 商品コード（SKU） |
| 発売日 | 発売日 |
| JANコード | ISBN |
| 配送グループ管理番号 | 配送パターン（値はそのままコピー）|

- 該当ヘッダーが無いタブでは、その項目は**空欄**で送る（例: グッズ/雑誌のISBN無し）
- 実装時に各タブの実ヘッダーを読み、候補名リストを最終確定する

## 重複・スキップ規則

- 送信先①商品入力シートの既存「商品コード」列を読み、**同じ商品コードがあればスキップ**（追記しない）
- ソース行の**商品コードが空の行はスキップ**（警告カウント）
- 同一実行内での重複も商品コードでスキップ

## 書込処理

1. `LockService.getDocumentLock()` で多重実行防止
2. 台湾系タブを横断し、A列チェックON行を収集（タブ名・行番号も保持）
3. 各行を5項目にマッピング → 送信候補リスト作成
4. 商品コード空 / 送信先重複 を除外
5. **確認ダイアログ**（件数＋先頭10件プレビュー、はい/いいえ）。いいえなら中止
6. ①商品入力シートの最終データ行（3行目以降の実データ最終行）の次から、
   ヘッダー名で特定した列へ `setValues` で一括追記
7. 成功した行の**A列チェックをFALSEに**（再送防止）
8. 完了トースト: `送信◯件 / 重複スキップ◯件 / コード無しスキップ◯件`

## エラー処理

- 送信先ブック/「①商品入力シート」が見つからない → `ui.alert` で中止
- ロック取得失敗 → 「他の処理が実行中です」トースト
- チェックON行が0件 → 「対象なし」トースト
- 送信先の必須ヘッダー（商品名/商品コード等）が見つからない → アラートで中止

## 既存パターンの踏襲

`Project_19/【大邱】データ転送.js` の同一ブック内転送を踏襲しつつ、
**別ブックは `SpreadsheetApp.openById` で開く**点が相違。
ロック・確認ダイアログ・プレビュー・トースト・最終データ行判定の作法は同様。
